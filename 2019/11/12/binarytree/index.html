Binarytree is a rather necessary part in data structure. Befor binary tree, let’s talk about tree

Tree

A tree consists of nodes and edges. Every node contains value and or its child node(if it has). Every tree has a root node, which is the first node(topmost) of the tree. e.g. Node “A” is the root of the following tree. If the root node is connected by another node, the root is the parent node of that node, and respectively, that node is child node of the root node.

Leaves are those nodes without child nodes, e.g. E,F,G,C,H are all leaves.

  • The depth of a node is the number of edges from the node to the tree’s root node.
    A root node will have a depth of 0.
  • The height of a node is the number of edges on the longest path from the node to a leaf.A leaf node will have a height of 0.

Properties of a tree:

  • The height of a tree would be the height of its root node,
    or equivalently, the depth of its deepest node.
  • The diameter (or width) of a tree is the number of nodes on the longest path between any two leaf nodes. The tree below has a diameter of 6 nodes.

Binary Tree

If each node of a tree has 2 child nodes at most, the tree is called binary tree. Binary tree is used often in practice. Two child nodes is called left child node and right child node according to their location. Now we are going to talk about binary search tree.

Binary Search Tree
