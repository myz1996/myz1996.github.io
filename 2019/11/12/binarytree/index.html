Binarytree is a rather necessary part in data structure. Befor binary tree, let’s talk about tree

Tree

A tree consists of nodes and edges. Every node contains value and or its child node(if it has). Every tree has a root node, which is the first node(topmost) of the tree. e.g. Node "A" is the root of the following tree. If the root node is connected by another node, the root is the parent node of that node, and respectively, that node is child node of the root node.

Leaves are those nodes without child nodes, e.g. E,F,G,C,H are all leaves.

  • The depth of a node is the number of edges from the node to the tree’s root node.
    A root node will have a depth of 0.
  • The height of a node is the number of edges on the longest path from the node to a leaf.A leaf node will have a height of 0.

Properties of a tree:

  • The height of a tree would be the height of its root node,
    or equivalently, the depth of its deepest node.
  • The diameter (or width) of a tree is the number of nodes on the longest path between any two leaf nodes. The tree below has a diameter of 6 nodes.

Binary Tree

If each node of a tree has 2 child nodes at most, the tree is called binary tree. Binary tree is used often in practice. Two child nodes is called left child node and right child node according to their location. Now we are going to talk about binary search tree.

Binary Search Tree

Traversals (Inorder, Preorder and Postorder)

Inorder

Inorder is the most common way for binary search tree. Inorder, which means leftchildtree, current node, rightchildtree.

Let’s look at a simple example, a tree only has 3 nodes.

  1. inOrder(A) inOrder(B) 1

    ​ visit A 2

    ​ inOrder(C). 3

  2. inOrder(B) inOrder(null)–return 4

    ​ Visit B 5

    ​ inOrder(null)–return 6

  3. inOrder(C) inOrder(null) –return 7

    ​ Visit C 8

    ​ inOrder(null)–return 9

As above, there are 3 main inOrder( ), but how about the exact Order of these code?

first, main method calls inOrder(A) as A is the root of the tree, A search its left child tree, so it calls inOrder(B). Now we jump to inOrder(B) method and we want to search B’s left child but B does not have left child so inOrder(B.leftChild) returns null. And B is visited. Go on, it’s time to call inOrder(B.rightChild) and it returns null as well.

At this time, A’s left child all finishes and A is visited. The same step is for Node C.

In conclusion, the step is 1,4,5,6,2,3,7,8,9. And the reuslt is B,A,C.

Preorder & PostOrder

Preorder: visit the current node itselt, traversing its left child tree , traversing its right subtree.

Postorder: traversing its left child tree, traversing its right subtree, visit the current node itselt.

Time complexity of binary trees

The tree is highly efficient for all common data storage operations. Tree’s traversal is not as fast as other operations, but in the database, traversal is not a basic operation.

Array Tree

Index of an array reflects the position of the node in the tree. 0 is root, 1 is the leftChild node of root.

Store each layer of the tree in order from left to right. If a node is index, its left child is 2 * index +1, right child is 2 * index + 2, parent is (index - 1) / 2.

Conclusion

Binary tree is a special kind of data structure with high efficiency, except for deleting. When deleting a node, there are various cases we must consider, which is comlipicated. Maybe we should avoid deleting a node actually.

Chinese Version

二叉树出现的频率较高，查找和插入比较简单。删除还是挺复杂的，第一遍看的时候云里雾里的。从根节点开始，首先得找到这个要删除的结点的位置，还要找到这个结点的父节点的位置，并设置一个布尔类型来保存要删除的结点在其父节点的左边还是右边以方便后面的操作。接下来就能删除了吗？这时出现了几种情况，需要分开考虑。

  1. 被删除的结点是叶子结点，这种情况是最简单的。只需要断开其父节点与其连接即可。但是假如要删除的结点是根节点呢？将root设为null,清空了整棵树。

  2. 被删除的节点只有一个子节点。当要删除节点的右子节点为空，只有左子节点。如果该节点是其父节点的左子节点，将父节点的左子节点连接到该节点的左子节点。如果该节点是其父节点的右子节点，将父节点的右子节点连接到该节点的左子节点。别忘了单独考虑该节点是否为root, 如果是则要把root的左子节点置为root。同理写出要删除的节点的左子节点为空，只有右子节点。按不同的可能性，将父节点与要删除的节点的子节点进行连接。

  3. 要删除的节点有两个子节点。这是最复杂的一种情况。首先要考虑的是，删除这个节点以后，用哪个节点来放在这个位置上呢？左子节点还是右子节点呢，会不会破坏二叉树的性质呢？这个时候出现了一个东西叫后继节点，即找到一个大于要删除的节点中最小的一个节点。怎么找，从当前的右边出发（右边一定都是大于当前节点的），然后拐到左边，一直向下走找到一个没有左子节点的节点即为后继节点。

    定义一个方法，getSuccessor(Node delNode)，用于找到后继节点。在这个方法里，找到后继节点以后并没有结束，如果后继节点不是要删除节点的右子节点，则首先要把后继节点的右子树全部连接到后继节点的父节点上。
    successorParent.leftChild = successor.rightChild;

    其次，要删除的节点的右子树也需要被处理。将要删除节点的右子树置为后继节点的右子树。

    successor.rightChild = delNode.rightChild;
