#Algorithm

Insertion sort is another kind of simple sorting method.

Starting from the second element of the array, each element in the array is inserted into the sorted array in ascending or descending order.
In general, the first element of the array is used as the starting element, we can assume that the first element has been in order and the second element should be inserted in order. Since the array to be inserted is already sorted, just sort the insertion point insertion element from right to left (or from back to front), and so on, until the last array element is inserted into the array, the process is complete.

e.g. {10,2,4,6,1,8}

{2,10,4,6,1,8} swap 2 & 10

{2,4,10,6,1,8} swap 4 & 10

Time Complexity

Insertion sort uses 􏰁N 2/4 compares and 􏰁N 2/4 exchanges to sort a randomly ordered array of length N with distinct keys, on the average. The worst case is 􏰁N 2/2 compares and 􏰁N 2/2 exchanges and the best case is N - 1 compares and 0 exchanges.

If an array has N x N elements, the best case for sorting is all elements below the diagonal should be swaped. For randomly ordered arrays, each item goes about halfway back, on the average. So the best case is that half of the entries below the diagonal.

An inversion is a pair of entries that are out of order in the array. Inversion Count for an array indicates how far (or close) the array is from being sorted. If array is already sorted then inversion count is 0. If array is sorted in reverse order that inversion count is the maximum.
Formally speaking, two elements a[i] and a[j] form an inversion if a[i] > a[j] and i < j.

Example:
The sequence 2, 4, 1, 3, 5 has three inversions (2, 1), (4, 1), (4, 3).

In E-X-A-M-P-L-E, 11 inversions(E,A),(X,A),(X,A),(X,M),(X,P),(X,L),(X,E),(M,L),(M,E),(P,E),(L,E)

If the number of inversions in an array is less than a constant multiple of the array size, we say that the array is partially sorted. Insertion sort is a rather efficient way for such arrays

The number of exchanges used by insertion sort is equal to the number of inversions in the array, and the number of compares is at least equal to the number of inversions and at most equal to the number of inversions plus the array size minus 1.

*Time Complexity: *O(n*2)

Auxiliary Space: O(1)

Boundary Cases: Insertion sort takes maximum time to sort if elements are sorted in reverse order. And it takes minimum time (Order of n) when elements are already sorted.

在冒泡排序之后复习总结了一下插入排序。插入排序可以分为直接插入排序，二分插入排序以及稍微有些难理解的希尔排序。

插入排序的思路是，将一组元素想象成未排序的和已经排序好的部分。要使这组元素全部有序，那么我们是不是应该把还没有序的部分一个一个插入到已经排好序的部分呢？可能有人就会问，我拿到一个全部随机全部无序的数组怎么办呢。按照国际惯例，把第一个元素当作已经被排好序的部分，一个元素可不就是被排好序了吗。接下来，从index = 1开始，一个一个把右边的元素与左边比较。

直接插入排序有一种优化的思路就是二分插入排序。当未排序的元素需要插入到左边时，不如用二分法来一步到位找到需要插入的位置。而不是一个一个一个比较。然后内层循环设置j>需要插入位置的index，将需要插入位置右边的数一个一个挪到右边。最后把空出来的坑填上array[j]。
