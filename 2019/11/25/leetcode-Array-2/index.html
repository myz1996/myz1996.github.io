#11 Container With Most Water

Two pointers, one from 0, the other from right.

这题我觉得难点在于想清楚，如何让指针移动。最大的面积有两种情况，长度最长即在一头一尾，或者底比较短但是高度很高。

我们开始把两个指针放在两边就是默认从第一种情况开始。然后根据木桶理论取短边的高度计算面积。当左边高度小于右边的高度时，左指针++找寻下一个可能的更高的。否则右指针👈移动一个。

#15 3Sum

res.add(Arrays.asList(new Integer[]{nums[i], nums[left], nums[right]}));

学习一下简便的写法。这题要注意的是去重，去重 = sort +

if(i != 0 && nums[i] == nums[i - 1]) continue;

定义两个指针，从头扫到倒数第二个，然后在i的右边通过两个指针寻找组合。在找到一个组合以后，还要注意的是，分别移动两个指针到不重复的地方。否则会有相同的解集出现。然后再正常各移动一个位置。

如果当前三个数不符合条件，则根据情况移动指针。

#16 3Sum Closest

这题虽然思路很简单，只要绝对值更小就进行更新替换。但是在实施的过程中，一是忘记了如果直接等于目标值，直接返回。而是这里与3sum不同的是，不需要两个while去寻找下个不同的元素。因为其实相同的值是可以被使用的。和上一题不一样的是，我们这里要找的是个最大值，而不是解法，那么最大值一定是有唯一性的。

另外与上一题不一样的是，上一题在找到一种解法以后，两个指针同时移动。这里不可以，因为只要不等于target，就会比target小或者大。也就是说只能移动一个指针。

#53 Maximum Subarray ???

这题虽然是简单题，但是却有很多很好的方法可以解决。

1: 动态规划

首先定一个变量，和为0，Max则为int 型最小值。方便后面的更新。

从第一个元素开始，一个一个把元素加到和上，如果和为负数，后面再加任何数一定没有后面那个元素的自身大。所以这个时候，我们应该先把和重新置为0。

2: 直观好理解的方法

第二种方法与动态规划比起来要好理解一点,循环遍历，只要当前的和加元素比元素独自本身大，就累积。否则将变量置为当前元素。

3:分治法

先找左半部分最大值，再找右半部分最大值。然后这两个值与跨越中间点的值进行比较。

有点没明白。。。递归真是我的烦恼。

#45 Jump Game II

这是理解的第一题Hard题，下午在家做的简直想。。。

用了两个循环，内层循环找到从当前元素到当前元素所能走到的最远元素过程中，每个元素所能跳到的最远元素。为什么？因为当前元素走到他能达到的最远距离时就不得不作出一个决定了。

这个时候步数加一。

我终于好不容易又理解了一种方法。

先看循环，从第一个元素开始判断当前元素所能达到的最远坐标和当前元素前的某个元素所能达到的最远坐标。如果当前元素比较大，那么就把next reach设置为当前的元素对应的值。第二个if判断是因为，前面的某个结点这个元素就不得不作出选择。我之前想，那不一定是这次跳啊，nonono，这一次代表了这个元素跳的次数。那么从第一个元素开始，已经跳了一次。从第一个元素到它所能达到的最远坐标的过程中，如果中间的元素没能让nextreach这个变量更新，那么它必须再跳一次。步数又加一。
