Balanced Search Trees

This kind of data structure is guaranteed to be logarithmic.

2-3 search trees

为了保证树的平衡性，允许一个节点保存多个键。现在引入3结点的情况。

原来的结点称为2结点，一个键及对应的值， 两条链接。左链接指向2-3树中的键都小于该结点，右链接指向的键都大于该结点。

3结点：含有两个键及对应的值和三条链接，左链接指向的2-3树中的键都小于该结点，中链接指向的键都位于该结点的两个键之间。右链接指向的键都大于该结点。

指向空树的链接称为空链接。

一棵完美平衡的2-3查找树中所有空链接到根结点的距离都应该是相通的。

Search

要判断一个键是否在树中，先讲它和根结点的键比较。如果是空链接则查找失败，其余情况根据比较的结果继续向下走。

Inserting a node

1.

直接插入可能无法保持树原来的平衡性，如果查找的结果位置在一个2结点，那么将其替换为3结点即可。

2.如果查找结果结束于一个3结点。

为了能将新键插入我们先临时将新键存入该结点中，使之成为一个4结点，3个键和4条链接。再将4结点转换为三个2结点。

现在这个小树是一个完美平衡的2-3树了，所有的空链接到根结点的距离相等，插入前高度为0，插入后高度为1。

3.向父结点是2结点的3结点插入新键

同样，先创造一个4结点，将原来父结点替换为一个新的3结点。这种变换是2-3树的动态变化的核心。此次变换仍然保持了树的有序与平衡。

4.向父结点为3-结点的3结点插入新键

再次构建一个临时4结点。想将中键插入父结点中，但很可惜父结点是个3结点。那么我们将这个中键构造一个新的4结点，在这个结点上进行相同的变换，分解这个父结点并将它的中键插入到它的父结点中。

即，一直向上分解临时的4结点直至遇上一个2结点并将它替换为一个不要分解的3结点或者是到达3结点的根。那么我们要怎么分解根结点呢？

分解根结点

从插入结点到根结点上全是3结点，根结点最终是一个临时4结点。那么只要把这个4结点分解为三个2结点即可。完美保持树的性质，树高加一。

总结四结点拆解的六种情况

在一棵大小为N的2-3树中，查找和插入操作访问的结点必然不超过lgN个。

在一个含有n个结点的2-3树的高度在log3N=lgN/lg3 和 lgN之间。

分别为（全是3结点）（全是2结点）。

由此可见，就算按照升序连续插入，2-3树也不会是一个最差的二叉树，而是一个高度为2的树。

因此，2-3树在最坏情况下性能也会很不错。每个操作中处理每个结点的时间都不会超过一个很小的常数。（10亿个结点的2-3树高度仅在19-30之间，最多只要访问30个结点就能够在10亿个键中进行任意查找和插入操作！惊了）




Red Black BST

红黑二叉树的基本思想是用标准的二叉查找树和别的信息来表示2-3树。由一条红色左链接相连的两个2结点表示一个3结点。黑链接则还是原来2-3树中的普通链接。

红黑树的定义：

  • 红链接均为左链接
  • 没有任何一个结点同时和两条红链接相连
  • 该树是完美黑色平衡的，任意空链接到根结点的路径上的黑链接数量相同。

满足这三个条件的二叉查找树和相应的2-3树就是对应的了。

颜色

每个结点都会有一个指向自己的链接，从父结点指向它。将链接的颜色保存在表示结点的Node的布尔变量color中。如果为红色，则为true，黑色则为false。空链接为黑。

旋转

左旋转：一条红色的右链接需要被转化为左链接

比较旋转前后可以发现，小于E 大于S的区间的顺序并没有发生变换。只是将根结点变为S，即右边的结点了。

右旋转：改变left为right即可。

rotateLeft()和rotateRight()都能返回一条链接。

向2结点中插入新键

一棵只含有一个键的红黑树只有一个2-结点，插入另一个键之后我们马上就需要将他们旋转。如果新键小于老健，我们只需要新增一个红色的结点，新的红黑树和单个3-结点完全等价。

如果新键大于老键，则会产生一条红色的右链接，此时需要

root = rotateLeft(root);

将其旋转为红色的左链接并修正根结点的链接。

向树底部的2-结点插入新键

插入新键将会在树的底部新增一个结点（保证有序性）。但总是用红链接和它的父结点相连。如果指向新结点的是父结点的左链接，父结点直接成为一个3-结点。如果指向新结点的是父结点的右链接，就需要旋转。

向一棵双键树（3-结点）中插入新键

  • 新键大于原树中两个键，连接到3-结点的右链接。将两条红链接都由红变黑，就得到一个三个结点组成的高为2的平衡树。
  • 新键小于原树中两个键，则会产生连续的两个红链接。将上层的红链接右旋。
  • 新键介于两个键之间，将下层的红链接左旋。

第二三种情况都要转换为第一种情况处理。

颜色转换

每次插入后都将根结点设为黑色。每当根结点由红变黑时树的黑链接高度就会加一。

向树底部的3-结点加入一个新结点，如果指向新结点的是3-结点的右链接则转换颜色即可。指向的是左链接则先右旋再转换颜色。指向的是中链接则先须先旋转下层链接然后右旋上层链接，最后转换颜色。颜色转换会使到中结点的链接变红，相当于将它送入了父结点。相当于在父结点中插入一新键。

红黑树的插入

删除

先了解一下2-3-4树，它的插入算法沿查找路径向下进行变换是为了保证当前结点不是4结点。向上变换是为了将之前创建的4结点配平。

在向下的过程中，如果遇到父结点为2-结点的4-结点，将4-结点分解为两个2-结点并将中键给他的父结点，使父结点变为一个3-结点。

如果父结点为3-结点，将4-结点分解为两个2-结点并将中键给父结点。使父结点变为一个4-结点。不用担心会遇到父结点为4-结点的4-结点。

  • 将4-结点表示为由三个2-结点组成的一棵平衡的子树，根结点和两个子结点都用红链接相连。
  • 向下的过程中分解所有4-结点并进行颜色变换。
  • 向上的过程中用旋转将4-结点配平。

2-3树删除最小键

从树底部3-结点删除一个键很简单，但是2-结点删除会留下一个空结点，破坏树的完美平衡性。

为了保证我们不会删除一个二结点，从左链接向下进行变换。确保当前结点不是2结点。根结点的情况：根为2-结点且两个子结点都为2-结点，将这三个结点变为4-结点。否则要保证根结点的左子结点不是2-结点。

保证以下情况之一成立即可：

  • 当前结点左子结点不是2-结点
  • 当前结点左子结点是2-结点而它的亲兄弟结点不是2-结点，将左子结点的兄弟结点中一个键移动到左子结点中。
  • 当前结点的左子结点和它的亲兄弟结点都是2-结点，将左子结点、父结点中的最小键和左子结点最近的兄弟结点合并为一个4-结点。使父结点由3-结点变为2-结点或者由4-结点变为3-结点。

在遍历的过程中执行这个过程，最后得到一个含有最小键的3-结点/4-结点。然后再回头向上分解所有临时的4-结点。

红黑树的性质

一棵大小为N的红黑树的高度不会超过2lgN

红黑树的最坏情况是它所对应的2-3树中构成最左边的路径结点全部都是3-结点而其余均为2结点。最左边的路径长度是只包含2-结点的路径长度（～lgN）的两倍。

一棵大小为N的红黑树中，根结点到任意结点的平均路径长度为～1.00lgN

一棵红黑树中，查找、插入、查找最小最大键、floor()、rank()、select()、删除最小最大键、删除、范围查询都是对数级别的。
