与33题很像只不过多了可以重复的值。二分法仍然可以用但是，3 1 1这种情况怎么办呢？既然target不为nums[mid]，而nums[mid]又等于nums[right],不如把right向左移动吧，这样子mid也会变，总能移动到一个nums[mid]和nums[right]不同的地方进行判断（如果这个数组有不同的元素的话）。

需要注意的是二分法，我是拿nums[mid]与nums[right]进行大小比较的，当然也可以和nums[left]进行大小比较。方法类似，但是与left比较的话，就要对left++操作，因为if,else if中nu ms[mid]和nums[left]都不符合，因为他们是相等的，而不要对right操作，条件比较不涉及right，如果对right进行移动，很有可能直接把这个target值给漂过。

此外还要注意这里：

#88 Merge Sorted Array

这题需要注意的地方是，要从后往前，否则会把nums1原来的值给覆盖掉。最后需要两个循环，防止某个数组没有添加完毕。

一种简单一点的方法：只要有一个数组没有到头就可以继续，其实是不用判断另一个>=0的，因为每次循环结束都会判断i >= 0 || j>= 0是否成立，必然有一个为正。这两个判断要写在前面防止空指针及数组越界。

#105 Construct Binary Tree from Preorder and Inorder Traversal

前序的第一个一定是根结点。到中序数组中找到根结点，左边的为左子树元素，右边即为右子树元素。

那么对于每一个左子树或者右子树来说，给他们相应的两个数组，他们就能找出根结点和他们的左右子树。本题的重点在于传递的两个数组的范围。

传给左子树的是，preorder第二个开始，因为第一个是根结点。传给右子树的是从第二个元素开始加上左子树的长度。左子树的长度可由根结点在inorder中的位置-inorder左边起始点得到。
