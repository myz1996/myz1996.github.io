(ALL IMAGES COME FROM GOOGLE IMAGE)

Binarytree is a rather necessary part in data structure. Befor binary tree, let’s talk about tree

Tree

A tree consists of nodes and edges. Every node contains value and or its child node(if it has). Every tree has a root node, which is the first node(topmost) of the tree. e.g. Node “A” is the root of the following tree. If the root node is connected by another node, the root is the parent node of that node, and respectively, that node is child node of the root node.

Leaves are those nodes without child nodes, e.g. E,F,G,C,H are all leaves.

  • The depth of a node is the number of edges from the node to the tree’s root node.
    A root node will have a depth of 0.
  • The height of a node is the number of edges on the longest path from the node to a leaf.A leaf node will have a height of 0.

Properties of a tree:

  • The height of a tree would be the height of its root node,
    or equivalently, the depth of its deepest node.
  • The diameter (or width) of a tree is the number of nodes on the longest path between any two leaf nodes. The tree below has a diameter of 6 nodes.

Binary Tree

If each node of a tree has 2 child nodes at most, the tree is called binary tree. Binary tree is used often in practice. Two child nodes is called left child node and right child node according to their location. Now we are going to talk about binary search tree.

Binary Search Tree

The left subtree contains node with lesser key than the node, the right subtreee contains node with greater key than the node or equal to the node.

GitHub Pages + Hexo/Hexo搭建GitHub静态博客

I always want to build my own website and after considering of Alibaba Cloud, i chose GitHub Pages because of its free price and simplicity.

But what is GitHub pages? Simply speaking, it provides us a free and easy way to publish and share our projects online. No database, no java web, just simple git. Althoug it is a static site hosting service, it is rather enough for me.

Let’s start!

1.Download

Prepare: Node.js ,download the latest version

​ Git

​ GitHub account

(I used Mac OS and i found that i cannot open the pkg of node.js, just right click it “open with”)

2.Install

Open terminal and insert:

$ sudo npm install -g hexo

and follow the tip to insert password

Now, let’s go a folder and new a folder for your GitHub Pages! For me, i new the folder /users/miao

$ hexo init blog

the blog folder contains all files about our GitHubpages. Let’s cd the blog folder and install npm:

$ npm install

It is time to check if we have successfully install hexo：

$ hexo s

This instruction means we open the hexo service, open our browser and enter the address: localhost:4000. If you can see the following pictures, congratulations!

3.Association

Sign in your GitHub account and create a new repository. Attention：the name of the repositoy must be yourusername.github.io

Here is the example, myz1996 is my username.

Slide to the bottom and modify these lines according to your own username

Save and back to blog folder insert step by step.

$ hexo g

$ hexo d

May be you will be faced with some problem here, check typo or install hexo-developer-git

`$ npm install hexo-deployer-git –save

Insert the following again:

$ hexo g

$ hexo d

Terminal asks you to enter the username and password of GitHub account. And what will happen? Enter the address http://yourusername.github.io. Wow! We can the the same page with http://localhost:4000.

We are almost done!

4.Add SSH

Generating public/ private rsa key pair, replace the email address with your GitHub account

$ ssh-keygen -t rsa -C "your_email@example.com"

Under .ssh folder we can see id_rsa and id_rsa.pub, this folder can be found in root. SSH is the content in id_rsa.pub, copy it. Go to GitHub, click settings, choose SSH keys and add SSH key. You can randomly enter the title, paste in key and click the button ‘add key’.

The use of SSH is we do not need to enter GitHub’s username and password every time. It becomes quite easy for us to post!

4.Themes

Hexo provides many free themes! You can go to Hexo themes to select your own. How to apply the new theme? First, cd “themes”, and git clone + the address of the new theme, take the following as an example:

https://github.com/iTimeTraveler/hexo-theme-hiero.git

This is the theme i currently use. Hold on, we still have to do one step:

$ vim _config.yml

change the theme into new name!

Is it okay? No, we have to clean the projects to generate new files corresponding with our new theme

$ hexo clean

$ hexo generate

$ hexo deploy

$ hexo generate equals to $ hexo g

$ hexo deploy equals to $ hexo d

Well, we have completed totally!

5.Post

How can we publish a post? It is really easy! Just cd blog and :

$ hexo new "name of your article"

This will create a .md file under /blog/source/_posts, all posts will be stored here. We can use Markdown to edit theses files and after modification, we should remember that , clean, generate and deploy.

One more thing, i encountered ond problem and the reason is that after ‘ : ‘ need space!

6.Conclusion

GitHub pages is much easier than develop a whole website. We can purchase some domain name to bind with our GitHub. This is our own free blog, we can decide layout, contents and in the future, there will be still something excellent to be explored!
